# 大四 - Java 学习路线

你现在正处在秋招如火如荼进行中的阶段，但如果你还几乎什么都不会，情况确实比较严峻，可以说是基本 GG 了，不管是秋招还是春招，对于你来说都很难找到工作，你的目标是毕业前的一年快速学习，并且在毕业后找到工作。

## 基础知识(一个半月)

推荐教程：[韩顺平学 Java](https://www.bilibili.com/video/BV1fh411y7R8/?spm_id_from=333.337.search-card.all.click)

基础课程是绝对不能省略的，因为 Java 基础是你后续所有学习的核心。

## 企业开发框架(一个月)

推荐教程：[黑马 JavaWeb](https://www.bilibili.com/video/BV1m84y1w7Tb/)

这一套课程里面有涵盖了找工作用的全部技术框架，还有数据库能怎么用，可以快速的把你培养成一个码农。

## Git(三天)

Git 本质上只是一个工具型技术，它的核心作用就是帮助你管理代码版本。所以现阶段不需要研究得特别深入，只要会上传代码、会分支管理、会合并代码、会解决冲突，就已经够用了，其他高级玩法可以暂时不用纠结。

需要掌握的内容包括：区分 Git 和 GitHub 的作用，理解工作区的概念，掌握分支的使用，以及常见操作命令，例如代码提交、推送、拉取、回退、重置；熟悉分支操作流程，能够进行代码合并并解决冲突；了解标签的使用，掌握 cherry-pick 的基本用法，以及对 Git Flow 有一个整体认识。

学习的时候，每一个命令都要亲自敲一遍，至少形成一个大概印象，达到“需要时能够查阅并使用”的程度即可。平时建议多用 Git 管理自己的代码，多往 GitHub 上提交项目，用得多了，自然就熟练了。

推荐课程：[尚硅谷 Git 教程](https://www.bilibili.com/video/BV1vy4y1s7k6/)

这门课比较短，并且内容也比较简单，所以直接看视频学习即可。

## Linux(一周)

学习 Linux 的原因很简单——你的项目最终是要部署到 Linux 服务器上的。它本质上就是一个操作系统，所以现阶段不用把它想得特别复杂。你只需要会对文件进行增删改查、会安装软件、会把项目部署上去并成功运行，其实就已经够用了。

说实话，没有必要为了“学习 Linux”而单独投入太多时间啃理论教程。最有效的方式，是直接动手实践，自己买一台阿里云服务器，安装一个 Linux 系统，然后把自己的项目部署上去。只要你能成功部署并访问到，那这一步就算真正掌握了。

推荐教程：[韩顺平 Linux](https://www.bilibili.com/video/BV1Sv411r7vd/)

这个课程基于 CentOS 版本。你可能听说过 CentOS 已经停止维护，但问题不大。你只是去体验和熟悉 Linux 的使用方式，它本质上是工具。学的是基本指令和操作逻辑，而不是某个具体发行版。只要掌握了常用命令和部署流程，后面就算换成别的 Linux 系统，也能很快上手。

## 求职

上面这些内容学完以后，立刻开始写简历——目标是写一份 牛逼的简历。简历上的技能不一定每一条你都完全掌握，但整体要看起来很扎实、有分量。

接下来就开始投递简历，参加面试，让面试官“拷打”你。每一次面试都是宝贵的经验积累，你会清楚自己的短板，知道哪些地方需要补充。

在这个过程中，千万不要间断学习。最优策略是：一边面试、一边学习，遇到不会的知识点立即补上，再继续下一个面试或项目。

如果经过努力还是找不到工作，那就直接考虑培训班，通过系统化的辅导来提升技能和求职能力。
