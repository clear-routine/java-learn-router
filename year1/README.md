# 大一 - Java 学习路线

如果你现在还是大一新生，对 Java 感兴趣，那你真的来对地方了。这个文档的目标很明确，用尽可能高效的方式，带你把 Java 学扎实，并最终找到一份相关的工作。

既然你刚上大学，可能之前也没有接触过编程，那我们就从最基础的“编程”这个概念讲起。所谓编程，说白了就是使用“编程语言”。语言是什么？你其实早就懂了 —— 从小到大，你和别人交流都是通过语言。现在的变化只是不再和人对话，而是和机器对话。这种“和机器对话的语言”，就是编程语言。

机器本身不会说话，你可以把它想象成一个不会开口、但能识字的“哑巴”。你写代码，就像写一张纸条；写完之后，把纸条递给它，它会按照纸条上的内容去执行。这，就是编程的基本过程。

所以编程并没有想象中那么神秘，也没有那么可怕。你要做的事情，其实就是学会用一种语言，写出一段能让机器看懂并执行的“文字”。至于代码写得好不好，就像高考写作文一样，有的人文笔流畅，有的人表达普通，这就是差距所在。对应到编程里，就是代码是否清晰、是否优雅。

但你不用一开始就追求“优雅”。大多数人都是普通人，我们平时说话并不会刻意堆砌华丽辞藻，只要表达清楚、对方能听懂就足够了。只有在特定场景下，才需要讲究修辞和表达技巧。编程也是一样。作为初学者，把自己当成一个刚开始学说话的孩子就好，不需要一开始就对自己要求到极致。

当然，这种放松只针对“写代码的熟练度”，而不是对概念的理解。基础概念一定要学得准确、扎实，就像小时候学拼音一样，每一个音、每一条规则都要弄清楚。只要概念打牢，后面写代码自然会越来越顺。

## 基础知识(三个月)

推荐教程：[韩顺平学 Java](https://www.bilibili.com/video/BV1fh411y7R8/?spm_id_from=333.337.search-card.all.click)

本阶段的主要学习内容包括环境相关的配置与工具使用，例如环境搭建、IDEA 开发工具的基本操作、新建项目、运行与调试、界面配置以及插件管理；同时系统学习 Java 基础语法，包括基本语法规则、数据类型、流程控制和数组等内容；在此基础上深入掌握面向对象核心知识，如面向对象思想、方法与方法重载、封装、继承、多态、抽象类、接口和枚举；此外，还需要学习常用核心内容，包括常用类、String、日期时间处理、集合类、泛型、注解、异常处理、多线程、IO 流以及反射，为后续更高阶的开发打下扎实基础。

这一阶段的目标是打牢 Java 基础，把语法、面向对象思想和核心 API 理解清楚，为后续框架学习做好准备。

由于韩顺平老师的课程主要基于 Java 8，而现在的新项目大多已经向 Java 17 靠拢，所以你还需要额外学习 Java 17 的新特性。

为了在巩固视频内容的同时完成版本升级，推荐直接阅读一本基于 Java 17 的经典书籍——《Java 核心技术·卷 I》。这本书在 Java 领域地位非常高，被不少人称为 Java 编程的“圣经”，讲解体系扎实、逻辑清晰。不过，书中与 GUI 相关的部分可以选择性跳过，现阶段无需花太多时间在这上面。

在阅读这本书时，一定要手动敲代码，不能只看不写。可以把自己整理的笔记交给 AI，让 AI 帮你出一些对应的编程练习题，然后自己慢慢写、慢慢练。只有通过不断动手，你才能真正把知识内化，而不是停留在“看懂”的层面。

## 数据结构与算法(一个月)

当你学完一门编程语言之后，想马上去做真实业务，其实是比较困难的。因为真正的业务开发，往往还需要框架、数据库以及各种组件配合，不是只会语法就能搞定的。

这个时候，反而是学习数据结构与算法的好时机。因为它只依赖一门语言就可以完成，不需要额外环境和复杂配置。现阶段的核心目标，是通过大量写代码，去真正体会“写代码”的感觉，同时系统掌握新的数据结构和算法知识。

数据结构与算法阶段的学习内容可以整体理解为几个大的板块，首先是数据结构本身，包括线性结构中的数组（字符串）、链表（单向链表、双向链表、循环链表）、栈、队列（普通队列、双端队列）；以及散列结构相关内容，如集合、映射和 BitMap；树结构部分包括二叉树、二叉查找树、多叉树、前缀树，以及堆（小顶堆、大顶堆）；图结构则需要掌握最短路径、并查集、最小生成树和拓扑排序算法等核心内容。其次是排序与查找算法，包括冒泡排序、快速排序、插入排序、希尔排序、选择排序、堆排序、归并排序、计数排序、桶排序、基数排序，以及有序表查找（二分查找）、线性表查找、树结构查找和散列表查找。再往上是经典算法思想的训练，如贪心算法、分治算法、动态规划、递归、回溯、枚举，以及搜索算法中的深度优先搜索（DFS）和广度优先搜索（BFS）。此外，还需要掌握字符串匹配相关内容，如 KMP 算法和前缀树，以及位运算和复杂度分析，包括时间复杂度与空间复杂度的理解与计算。

推荐课程：[尚硅谷数据结构与算法](https://www.bilibili.com/video/BV1E4411H73v/?spm_id_from=333.337.search-card.all.click)

这门课程虽然看起来有点老，但数据结构和算法这些内容本身变化并不大，很多核心思想这么多年基本没变，所以直接学完全没问题。

不过一定要记住一点：必须自己动手写代码。每一个数据结构、每一道练习题，都要亲自敲一遍。只看不写基本等于没学，真的不写，结果大概率就是直接“gg”。

## 计算机网络(一个月)

想要学习项目开发，虽然你代码只是写后端代码，但是你写的后端代码是需要和前端配合的，配合走的就是网络，并且后端服务之间的相互调用也是走的网络，所以你必须要学习计算机网络。

本阶段需要掌握的知识主要包括网络分层模型及其各层的作用，理解一次完整的网络传输过程是如何进行的；掌握 IP 地址与端口的基本概念；熟悉 HTTP 与 HTTPS 协议的工作机制，以及 UDP 和 TCP 协议的特点与区别；了解 ARP 地址解析协议在局域网通信中的作用；同时对网络安全的基本原理有所认识，并理解 DNS 域名解析的整体流程。

推荐课程：[计算机网络微课堂](https://www.bilibili.com/video/BV1c4411d7jb/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

这门课也是比较老的，但是计算机网络也是万年不变的东西，并且这门课内容不算特别的复杂，对于入门小白非常的友好。

当然，视频看完之后，如果你还想进一步加深理解，也可以通过阅读书籍来补充知识。比如《图解 HTTP》，属于非常经典的入门读物；《网络是怎样连接的》，内容讲解通俗易懂，适合建立整体认知；以及《图解 TCP/IP》，更适合深入理解底层网络协议原理。这几本书我都看过，质量都很不错，有时间的话都可以读一读。

## MySQL 数据库(两个月)

在企业开发中，大部分业务数据都是通过关系型数据库来存储的，因此数据库是后端开发人员必须掌握的核心技能。其中，MySQL 是目前最主流的数据库之一，也是面试中的重点考察内容，必须认真对待。

学习内容方面，需要掌握数据库的基本概念和 MySQL 的搭建方式，重点熟练编写 SQL 语句，包括 DDL（数据定义语言）、DML（数据操作语言）、DQL（数据查询语言）和 DCL（数据控制语言）。同时要理解各种约束，例如主键约束、外键约束、唯一约束和非空约束。索引是重中之重，需要掌握索引类型、索引原理以及索引优化思路；事务同样必须掌握，包括 ACID 特性和隔离级别。锁机制建议学习，例如行锁与表锁、乐观锁与悲观锁。此外，还需要具备设计数据库表的能力，包括理解三大范式、掌握表关联设计，以及具备一定的性能优化意识。

数据库是后端开发的基石，一定要学扎实。尤其是 SQL 语句编写能力和数据库表结构设计能力，这两个能力必须真正掌握。比如让你设计一个学生管理系统，你应该能够清晰地规划出需要哪些表，例如学生表、班级表；每张表需要哪些字段，以及字段的数据类型；表与表之间是什么关系，是一对多还是多对多。这些看似基础的问题，实际上很多同学在真正做项目时都会感到迷茫，所以一定要提前练熟练透。

推荐课程：[黑马程序员数据库](https://www.bilibili.com/video/BV1Y7411K7zz/?vd_source=87e9b301b5680ecec2487ab62d311d4a)

这门课程对于初学者来说已经完全够用，只要你能坚持从头到尾认真听完，并跟着动手练习，一定会有明显的提升。不要三天打鱼两天晒网，踏踏实实学完，收获绝对不会小。

## Java Web(15天)

Java Web 阶段主要学习 Java 网页应用开发的基础内容，包括一点点前端基础知识、XML、JSON、Servlet、Filter、Listener、JSP、JSTL、Cookie 以及 Session 等核心内容。这一部分的意义，不是让你成为前端高手，而是为后续学习框架打好地基。因为很多框架本质上都是在这些技术之上一层层封装出来的，如果底层原理不清楚，后面学框架的时候就会很迷糊。

推荐课程：[黑马程序员 JavaWeb](https://www.bilibili.com/video/BV1Qf4y1T7Hx/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

跟着课程系统学一遍，理解原理并动手练习，后续学习 Spring 等框架时会轻松很多。

## 瑞吉外卖(两个月)

其实，上面这些技术如果你都系统学完了，找一份工作基本是没什么大问题的。但有一点必须清楚——只学习、不做题，在任何阶段都属于“假努力”。知识如果不通过实践巩固，很快就会变得模糊。

所以，你一定要自己完整做一个项目，把学过的东西真正用起来。

推荐项目：[瑞吉外卖](https://www.bilibili.com/video/BV13a411q753/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

这个项目在网上被不少人吐槽，甚至有点“烂大街”的感觉。但我想告诉你一个现实，大学生本来就很难接触到真正复杂的大型商业项目。你做项目的目的，也不是为了在面试时炫技，更不是为了显得多牛。

你做项目的唯一目的，是把学过的技术真正掌握住。它只是一个练习载体，仅此而已。这辈子你要做的项目多到数不过来，根本没必要在这里过度纠结“项目够不够高级”。能把一个项目完整跑通、独立写完、彻底理解，这才是最重要的。

## Git(三天)

Git 本质上只是一个工具型技术，它的核心作用就是帮助你管理代码版本。所以现阶段不需要研究得特别深入，只要会上传代码、会分支管理、会合并代码、会解决冲突，就已经够用了，其他高级玩法可以暂时不用纠结。

需要掌握的内容包括：区分 Git 和 GitHub 的作用，理解工作区的概念，掌握分支的使用，以及常见操作命令，例如代码提交、推送、拉取、回退、重置；熟悉分支操作流程，能够进行代码合并并解决冲突；了解标签的使用，掌握 cherry-pick 的基本用法，以及对 Git Flow 有一个整体认识。

学习的时候，每一个命令都要亲自敲一遍，至少形成一个大概印象，达到“需要时能够查阅并使用”的程度即可。平时建议多用 Git 管理自己的代码，多往 GitHub 上提交项目，用得多了，自然就熟练了。

推荐课程：[尚硅谷 Git 教程](https://www.bilibili.com/video/BV1vy4y1s7k6/)

这门课比较短，并且内容也比较简单，所以直接看视频学习即可。

## Linux(一周)

学习 Linux 的原因很简单——你的项目最终是要部署到 Linux 服务器上的。它本质上就是一个操作系统，所以现阶段不用把它想得特别复杂。你只需要会对文件进行增删改查、会安装软件、会把项目部署上去并成功运行，其实就已经够用了。

说实话，没有必要为了“学习 Linux”而单独投入太多时间啃理论教程。最有效的方式，是直接动手实践：自己买一台阿里云服务器，安装一个 Linux 系统，然后把自己的项目部署上去。只要你能成功部署并访问到，那这一步就算真正掌握了。

推荐教程：[韩顺平 Linux](https://www.bilibili.com/video/BV1Sv411r7vd/)

这个课程基于 CentOS 版本。你可能听说过 CentOS 已经停止维护，但问题不大。你只是去体验和熟悉 Linux 的使用方式，它本质上是工具。学的是基本指令和操作逻辑，而不是某个具体发行版。只要掌握了常用命令和部署流程，后面就算换成别的 Linux 系统，也能很快上手。

## 进阶学习

上面这些内容如果你都踏踏实实学完了已经过去一年了，其实已经完全具备工作的能力了。能力强、进度快的同学，可以直接去找实习，越早进场越好。

如果暂时不想实习，也没关系，那就继续打磨自己。把你能找到的优质免费课程尽可能多敲几遍，B 站上那些热门的“外卖项目”都可以拿来练手。反正基本都是单体项目，结构清晰，很适合强化训练。

不要觉得重复没意义，也不要嫌项目“烂大街”。真正能拉开差距的不是项目有多新，而是你敲了多少遍、理解得有多透。写得越多，踩的坑越多，你对整个开发流程的理解就会越深刻。

当然，上面的这些内容，其实也只是刚刚迈进工程师的大门而已。计算机已经高速发展了这么多年，领域广阔、技术繁多，还有大量知识等待你去探索。甚至可以说，就算学一辈子，也不可能把所有东西学完。

所以，一定要养成终身学习的习惯。技术在更新，你也要持续升级自己。更何况你可能还有两年的时间，可以用来沉淀和打磨自己，这段时间如果认真投入，成长会非常明显。

接下来，我会罗列一些你还需要继续深入学习的技术方向。

### 设计模式(一个月)

设计模式，是软件开发中针对某一类问题的通用解决方案。学会使用设计模式，可以让你的代码更加优雅、清晰、易扩展、易维护。也正因为如此，很多框架的源码中都大量运用了设计模式，如果你没有相关基础，读源码时会非常吃力。

常见的设计模式包括创建型模式，如单例模式、工厂方法模式、抽象工厂模式、建造者模式、原型模式；结构型模式，如适配器模式、组合模式、装饰器模式、代理模式、享元模式、外观模式、桥接模式；以及行为型模式，如迭代器模式、模板方法模式、策略模式、命令模式、状态模式、责任链模式、备忘录模式、观察者模式、访问者模式、中介者模式、解释器模式等。

不过要记住，设计模式不需要死记硬背。它们本质上是一种“把代码写得更好”的思路，而不是考试背诵题。你可以把它理解为写代码时的一种表达能力提升。还记得之前说过编程语言就像我们写字一样吗？学好设计模式，就像一个作家学会了更丰富的写作技巧，能够写出更有层次、更优雅的文章。道理其实很简单。

推荐课程：[尚硅谷 Java 设计模式](https://www.bilibili.com/video/BV1G4411c7N4/?vd_source=87e9b301b5680ecec2487ab62d311d4a)

这门课程系统讲解了常见的设计模式，通过实例演示帮助你理解每种模式的用途和实现方式，一定要自己敲一遍！

### redis(半个月)

Redis 和 MySQL 本质上都是数据库，都是用来存数据的，只是侧重点不同。MySQL 更偏向持久化的关系型数据存储，而 Redis 是基于内存的高性能数据库，常常被拿来做缓存或者解决高并发问题。

你需要掌握的核心内容包括：Redis 基础、什么是缓存、本地缓存、Caffeine 库、多级缓存、Redis 分布式缓存、常见数据类型、常用操作命令，以及在 Java 中如何操作 Redis，比如使用 Spring Boot 的 RedisTemplate、以及更高级一点的 Redisson。

进阶部分包括主从模型搭建、哨兵集群搭建，以及持久化机制（RDB 和 AOF），理解这些是为了让你知道 Redis 在生产环境中是怎么保证数据安全和高可用的。

在实际项目中，Redis 的应用场景非常多，比如数据共享、单点登录、计数器、限流、点赞、实时排行榜、分布式锁等。同时还要掌握缓存常见问题的处理方式，例如缓存雪崩、缓存击穿、缓存穿透以及缓存更新一致性问题。

学会简单使用 Redis 并不难，本质上和数据库类似，无非就是通过 API 做增删改查。但真正要用好 Redis，还是需要多结合实际场景去思考，比如哪些数据适合放在缓存里，缓存和数据库如何配合使用。

推荐你去看 黑马程序员 的 [Redis 教程](https://www.bilibili.com/video/BV1cr4y1671t/?vd_source=87e9b301b5680ecec2487ab62d311d4a)，这套课程讲得很清晰，而且是结合实际项目去讲，而不是纯理论。看完之后可以回头改造一下你之前做过的项目，比如给瑞吉外卖加上缓存，把高频查询的数据放进 Redis 里，你的理解会直接上一个台阶。


### 消息队列(一个月)

消息队列本质上是一个用来传输和保存消息的容器，在大型分布式系统里非常常见。它主要解决三个核心问题：应用解耦、异步处理、流量削峰。所以在后端开发中几乎是必学内容，也是面试里的高频考点。

你需要掌握的知识包括：消息队列的作用原理，生产者–消费者模型，交换机模型，死信队列，延迟队列，消息持久化机制，Java 如何操作消息队列，以及基本的集群搭建能力。别只停留在“能发消息”，要理解它在系统架构里的位置。

消息队列本质上是一个组件，不是每个项目都会用到，但只要进入公司，大概率会接触到相关系统。所以既然大学时间充足，完全可以提前学扎实一点，不然工作中会非常被动。

现在更推荐直接学习 RocketMQ，它在国内企业中使用更广泛。如果时间允许，也可以学习一下 RabbitMQ，理解不同消息队列的设计思路，对架构视野很有帮助。

课程方面可以参考 黑马程序员 的 [RabbitMQ 教程](https://www.bilibili.com/video/BV1Xm4y1i7HP/?vd_source=87e9b301b5680ecec2487ab62d311d4a)，讲得比较清晰，适合入门；以及 动力节点 的 [RocketMQ 课程](https://www.bilibili.com/video/BV1jL41187ny/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)，对原理和实战讲解都比较深入。跟着完整敲一遍，比只看概念强得多。

### Nginx(一个月)

Nginx 是目前主流的开源高性能 HTTP 和反向代理服务器，常用于网站部署、请求转发、负载均衡、网关路由等场景。不管是前端还是后端，只要涉及到部署和服务器环境，基本都绕不开它，后端面试里也偶尔会问到。

你需要掌握的内容包括：Nginx 的核心作用、正向代理、反向代理（尤其是负载均衡）、常用命令、基础配置方式、动静分离（前后端分离项目部署）、以及简单的集群搭建思路。重点不是把配置全背下来，而是知道每一块配置在解决什么问题。

建议可以自己买两三台云服务器，实际搭建一下反向代理和负载均衡环境。比如一台作为入口服务器，后面挂几台应用服务，自己动手配置 upstream，看请求如何被分发。真正跑起来一次，比看十遍教程都管用。

推荐课程可以看 尚硅谷 的 [Nginx 教程](https://www.bilibili.com/video/BV1yS4y1N76R/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)，讲得比较系统，适合从零开始理解整体结构。把配置文件多改几遍，你会慢慢发现，其实 Nginx 本质上就是在“接请求 → 转请求 → 分请求”，没那么神秘。

### 微服务(两个月)

前面讲的那些项目基本都是单体架构，一个应用就是一个整体，部署也是整体部署。但在真实的大型项目里，往往会采用微服务架构——把一个庞大的系统拆分成多个小服务，每个服务负责一块业务，然后通过网络进行通信。拆分只是第一步，难点在于怎么让这些小服务之间稳定、高效地协作，这就涉及到一整套微服务技术体系。

学习微服务相关组件时，建议先学会“怎么用”，跑通流程，能搭建起来、能调用成功，再慢慢深入理解原理。不要一上来就钻源码，不然很容易劝退。

可以参考 [黑马程序员](https://www.bilibili.com/video/BV1kH4y1S7wz/?vd_source=87e9b301b5680ecec2487ab62d311d4a) 的微服务课程，这套课适合入门，能帮你把常见组件串起来，建立一个整体概念。

不过要提醒你，微服务组件非常多，课程只能带你入门。真正想吃透，必须自己做一个完整的微服务项目，在实践中搞清楚哪些组件是核心常用的，比如注册中心、配置中心、网关、消息中间件、分布式事务等等。等你踩过坑，再回头去看官方文档，那时候才能真正看懂。到了进阶阶段，想完全靠视频学明白几乎是不现实的，最终还是要靠自己查文档、做项目、踩问题。这个阶段会很痛苦，但也是能力真正拉开的阶段。

学完前面那一整套技术栈之后，不要再继续“囤课”了，直接开干，做一个完整项目，把所有东西串起来。

可以去做 [黑马头条](https://www.bilibili.com/video/BV1Qs4y1v7x4/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a) 这个项目。它本质上是一个综合性项目，涉及微服务、缓存、消息队列、搜索、权限、部署等内容，刚好能帮你把前面学的东西真正落地。

再强调一次，项目是不是烂大街真的不重要。面试官不会因为你做了“热门项目”就给你加分，他真正关心的是，你到底懂不懂原理？有没有自己踩过坑？出现问题会不会排查？

项目只是你练手的沙袋，不是用来炫耀的。你做它的目的只有一个——把技术打扎实。等你真正做通一两个完整项目，你对系统的理解会完全不一样。那种“原来这些组件是这样配合的”的感觉，比再看十门课都强。

### Docker(半个月)

使用容器技术的好处真的很多。它可以把应用和运行环境一起打包封装，实现环境隔离、独立部署，不再依赖“这台机器装了什么那台机器没装什么”。尤其是团队协作和服务器部署阶段，容器化几乎是标配。

你需要掌握的内容包括：Docker 基础概念、容器和镜像的区别、如何拉取镜像、运行容器、部署服务、编写 Dockerfile、使用 Docker Compose 管理多容器、了解 Docker Machine 和 Docker Swarm，以及多阶段构建这些常见技巧。核心目标不是背命令，而是知道如何把一个项目完整地容器化。

Docker 最大的价值，就是解决“在我电脑上能跑，在服务器上却跑不起来”的问题。环境被写进镜像里，一次构建，到处运行。部署流程会变得非常清晰，也非常稳定。

建议跟着视频教程，把基础操作全部自己动手敲一遍：拉镜像、启动容器、写一个简单的 Dockerfile，再把你的 Spring Boot 项目打成镜像跑起来。真正运行成功那一刻，你会明显感觉到部署思路变清晰了。

说到底，Docker 只是一个工具，让部署变简单。不要把它学复杂，你只需要做到一件事：让你的服务能在容器里稳定运行，那就算学懂了。可以参考 [尚硅谷](https://www.bilibili.com/video/BV1Zn4y1X7AZ/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a) 的 Docker 课程，讲得比较系统，适合打基础。

学完之后一定要回头，把你之前做的项目——无论是单体架构还是微服务——全部用 Docker 重跑一遍。单机跑、Compose 跑、多容器一起跑，自己折腾一下。这个过程才是真正把技术变成能力。

### 项目实战

把前面那一整套进阶内容啃完，时间基本也来到大二下学期了。走到这一步，其实你已经和刚入门时完全不一样了。接下来最重要的一件事只有三个字：多做项目，多做项目，多做项目。

建议你再花半年的时间，把市面上那些机构公开的免费项目尽可能都拿来做一遍。但注意，这个阶段的“做项目”和以前不一样了，不能再是看一段视频写一段代码的跟练模式。

你要开始学会分析需求。看视频之前，先自己梳理功能点，尝试写一份简单的需求文档，哪怕只有几页也行。把模块怎么划分、接口怎么设计、表结构怎么设计，都提前想清楚。然后关掉视频，自己纯手写接口和代码，实现完之后再对比老师的方案，看看差距在哪里。

尽量按照真实开发流程来做：接口写好，提供给前端调用；参数校验、异常处理、日志记录、接口规范，都要往“工程化”方向靠拢。慢慢让自己从“完成作业的学生”转变成“能交付系统的工程师”。

这个阶段会比之前累很多，因为你不能再依赖讲解，而是要独立思考。但恰恰就是这段时间，决定你和普通跟课选手的差距。半年时间沉下心来狠狠干一波，你的技术自信会完全不一样。

如果你准备进入“疯狂做项目”的阶段，下面这些公开视频项目都挺有代表性，拿来练手非常合适。别挑，能啃几个算几个。
