# 大一 - Java 学习路线

如果你现在还是大一新生，对 Java 感兴趣，那你真的来对地方了。这个文档的目标很明确，用尽可能高效的方式，带你把 Java 学扎实，并最终找到一份相关的工作。

既然你刚上大学，可能之前也没有接触过编程，那我们就从最基础的“编程”这个概念讲起。所谓编程，说白了就是使用“编程语言”。语言是什么？你其实早就懂了 —— 从小到大，你和别人交流都是通过语言。现在的变化只是不再和人对话，而是和机器对话。这种“和机器对话的语言”，就是编程语言。

机器本身不会说话，你可以把它想象成一个不会开口、但能识字的“哑巴”。你写代码，就像写一张纸条；写完之后，把纸条递给它，它会按照纸条上的内容去执行。这，就是编程的基本过程。

所以编程并没有想象中那么神秘，也没有那么可怕。你要做的事情，其实就是学会用一种语言，写出一段能让机器看懂并执行的“文字”。至于代码写得好不好，就像高考写作文一样，有的人文笔流畅，有的人表达普通，这就是差距所在。对应到编程里，就是代码是否清晰、是否优雅。

但你不用一开始就追求“优雅”。大多数人都是普通人，我们平时说话并不会刻意堆砌华丽辞藻，只要表达清楚、对方能听懂就足够了。只有在特定场景下，才需要讲究修辞和表达技巧。编程也是一样。作为初学者，把自己当成一个刚开始学说话的孩子就好，不需要一开始就对自己要求到极致。

当然，这种放松只针对“写代码的熟练度”，而不是对概念的理解。基础概念一定要学得准确、扎实，就像小时候学拼音一样，每一个音、每一条规则都要弄清楚。只要概念打牢，后面写代码自然会越来越顺。

## 学习路线

### 第一阶段：基础知识

首先，在 B 站上找一门系统的视频课程，从头到尾完整看一遍。中途不要做笔记，也不要跟着敲代码，但必须保证老师讲的每一个知识点你都真正听懂了。如果有听不懂的地方，就反复回放，直到彻底理解为止。

视频课程可以参考黑马程序员的课程，我当时看的是韩顺平老师的，但是感觉课程数量太多，不太适合追求速通的人。

当你把视频看完之后，其实已经对 Java 有了一个整体性的认识，但这还远远谈不上真正掌握。想靠视频课程彻底学会 Java，基本是不现实的。接下来要做的事情只有一个——看书。我推荐《Java核心技术·卷I》，这本书在 Java 领域非常经典，被很多人称为 Java 编程的“圣经”，内容讲解非常扎实、系统。

从这个阶段开始，你需要认真做笔记。但做笔记不是把书上的内容抄一遍，而是理解之后，用自己的话重新表达出来，然后反复记忆。只有这样，知识才真正属于你。同时，一定要配合写代码练习。建议使用电子版书籍，遇到不理解的地方可以直接整理问题复制给 AI，让 AI 帮你解释。

需要注意的是，这个阶段的核心任务是搞清楚语法和基础概念。书里的 API 可以暂时跳过，不用过分纠结。练习题可以让 AI 帮你出题，然后自己独立完成。但一定要记住，不要拿算法题当主要练习内容。这个阶段如果直接上算法，很容易把自己练到怀疑人生，从入门直接到放弃。

书籍一定要选择基于 Java 17 版本的。现在很多新项目都在逐步向 Java 17 靠拢，它已经成为主流的长期支持版本之一。早点接触这个版本，对你以后找工作和参与实际项目都会更有帮助。

耗时：2 个月

> 注意：Swing 用户界面组件、图形用户界面程序设计相关章节可以跳过，不必阅读。

> 如果你发现自己确实看书看不进去、也理解不了，那就先别硬撑着啃书。先把视频课程认真学好，比什么都重要。找一套系统的视频，好好做笔记，老师讲到的每一段代码、每一个课后作业，都要跟着完整敲一遍，不能只看不写。一定要同步实操，这样理解才会更牢。推荐可以去 B 站搜索韩顺平老师的课程，跟着系统学一遍。


### 第二阶段：夯实基础

计算机网络，直接看王道考研的 408 课程就行，书和视频配套着看，不一定非要买最新版，能找到电子版资源的话也可以先用着。这个阶段一定要尽量学懂，尤其是计算机网络，很多内容在后面都会反复用到。

耗时：1 个月

数据结构，建议找一套 Java 版本的课程来学，B 站上有不少相关资源，跟着系统学一遍就行。学习数据结构时，一定要用 Java 手动完整地敲代码，不要只看理论。光看不写是记不住的，只有自己一行一行敲过，理解每个细节，知识才会真正变成自己的。

耗时：1 个月

### 第三阶段：MySQL 数据库

随便找一套 MySQL 的基础教程，跟着完整学一遍就行。还是老规矩，不需要事无巨细地做笔记，但一定要真正掌握核心内容。

至少要做到，能够熟练写出最基本的增删改查（CRUD）语句，并且理解“联表查询”到底是怎么回事，知道表和表之间是如何通过字段建立关联的。

至于更深入的内容，比如性能优化、索引调优、复杂事务等等，这个阶段可以先不用纠结，先把基础打牢再说。

耗时：7 天

### 第四阶段：项目开发

接下来可以借助 AI 辅助你完成一个简单的实战项目。做一个基于 MySQL 联表的小项目，不需要设计得多复杂，核心目标只有一个，就是实现最基本的增删改查功能。

你需要做到的是：通过自己写的程序，把数据成功存入 MySQL 数据库，并且能够从数据库中正确读取出来。整个流程必须打通，从程序到数据库，再从数据库返回数据，每一步都要亲自跑通。这个项目的意义不在于“炫技”，而在于让你真正理解程序和数据库是如何配合工作的。只要这一点打通，你就已经迈过了一道非常关键的门槛。

> 在这个阶段，你真正需要训练的是“组件之间如何配合”。比如我现在可以直接告诉你：你需要用到 JDBC 这个技术。但问题是，如果你是完全自学，你一开始根本不会知道“哦，我应该用 JDBC”。那怎么办？这就涉及到一个很重要的能力 —— 学会提问。当你准备做这个项目时，你应该先问一个问题：Java 项目要怎么和数据库进行数据交互？需要用到哪些组件？当你问出这个问题之后，自然就会知道需要学习什么，比如 JDBC，然后再去专门学习这个技术，最后再回过头来把项目完成。这个过程本身，比项目结果还重要。因为你训练的是一种能力，当你遇到不会的事情时，知道如何拆解问题，知道该问什么，知道下一步应该学什么。这样以后你面对新的技术、新的需求时，才能真正独立思考，而不是一有不会就去翻 B 站找一整套现成课程，从头看到尾。

耗时：3 天

### Java Web

Java Web 这一阶段，直接找一套系统课程，尽快完整看一遍。还是一样，不需要事无巨细地做笔记，重点抓核心知识点：Servlet、Filter、Listener、Session、Cookie 这些一定要搞清楚原理和作用。

视频看完之后不要停，立刻开始写代码。把你之前写的那个带 Main 方法的程序，改造成一个真正的 Web 项目。目标很明确：你必须能够在前端页面上看到后端返回的数据，这一步成功了，才算真正学会。

至于前端页面，不用纠结样式和细节，可以让 AI 帮你生成基础页面代码。你的核心任务不是写漂亮页面，而是打通“浏览器 → 服务器 → 数据库 → 服务器 → 浏览器”这一整条链路。只要这条链路跑通，你的理解就会提升一个层级。

### 第五阶段：Spring 系列框架

Spring、SpringMVC 这两个框架，还是一样的思路，找一套系统课程，尽快完整看一遍，不需要事无巨细地做笔记。看完之后，直接学习 MyBatis，同样用“快速过一遍”的方式，不纠结细节。

当这三个框架都看完后，直接去 B 站找一个最简单的 SSM 项目。技术栈只需要包含 Spring、SpringMVC、MyBatis 三个框架即可。先完整看一遍视频，理解整体开发流程和代码结构，然后把后端部分自己从头到尾手动写一遍。写完之后直接运行，看看能不能成功跑起来。

在这个过程中，你一定会遇到一些不属于 SSM 体系的组件。不要慌，这很正常。既然是自学，那遇到新组件就顺带学一下。第一步先弄清楚它叫什么名字；第二步让 AI 帮你分析项目源码，搞清楚它在整个项目里的作用；理解清楚之后直接照着写进项目里。暂时不要扩展性学习，不要横向发散，当前阶段的核心只有一个，把 SSM 跑通。

还有一个非常关键的点：千万不要想着把整个项目一次性写完再启动。那样基本等于自杀式开发。正确的方式是——每写完一个接口，就启动一次项目测试是否正常；确认没问题，再写下一个接口。一直这样循环，直到所有接口都完成。这就是最小单元测试的思路。如果所有接口都写完才统一启动，问题会成倍增加，到时候排错成本会非常高。

耗时：2 个月

### 第六阶段：Spring 系列框架进阶

Spring 系列框架，可以说是在整个程序员职业生涯中极其重要的一项核心技术。因此，这一阶段一定要多花时间深入掌握。

前面你已经通过视频课程入门，也通过 SSM 项目做过实战铺垫，现在不再是“会用”就行，而是要开始系统、全面地学习。这一阶段就是纯粹的技术打磨，没有捷径。

如果你有一定的阅读能力，建议直接学习官方文档，理解框架设计思想和底层机制；如果暂时阅读英文文档比较吃力，也可以结合我的中文学习笔记循序渐进。

这一部分必须认真做笔记，而且要做详细笔记。每学一个技术点，都要通过一个小 demo 去验证。不要只是“看懂”，而是要“跑通”。只有真正写过、调试过，你对 Spring 的理解才会牢固。

到了这里，你已经不是在学“怎么写代码”，而是在学“框架是如何帮你管理代码的”。只要这一层理解建立起来，你的能力会发生质的变化。

耗时：2 个月

### 第七阶段：SpringBoot 框架

前面你已经有了非常扎实的 Spring 和 SpringMVC 基础，现在可以开始学习 SpringBoot 了。

同样的先找一个视频课程过一遍，不需要事无巨细地做笔记，前面你不是做了一个 SSM 项目吗，直接改成 SpringBoot 项目，然后运行起来，看看是否能够成功。

耗时：1 个月

### 第八阶段：SpringBoot 框架进阶

SpringBoot 会伴随你的开发生涯很多年，和前面一样你必须看官方文档，理解框架设计思想和底层机制，然后通过小 demo 去验证。

耗时：2 个月

到这里了其实你的已经具备工作能力了，至少最基本的工作能力你是有的，所以你完全可以自信起来了。

### 第九阶段：Mybatis 框架

前面看过 Mybatis 框架的视频，现在直接看官方文档，理解框架设计思想和底层机制，然后通过小 demo 去验证。

### 第十阶段：MySQL 高级

MySQL 高级阶段的学习，还是先找一套系统的课程，尽快完整看一遍。不需要事无巨细地做笔记，重点是把整体架构和关键技术理解清楚。

看完课程后，再深入研读一本书 —— 《深入浅出 MySQL：数据库开发、优化与管理维护（第3版）》。这本书适合用于提升你的优化思路、性能调优技巧和数据库管理能力，是进阶 MySQL 的经典读物。

耗时：2 个月
