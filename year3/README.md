# 大三 - Java 学习路线

先给你一个大致的时间概念，如果你打算参加秋招，其实你大概只剩 10 个月左右的时间。因为每年从 7 月份开始，秋招就会陆陆续续启动。

这也就意味着，你的学习时间其实是非常紧张的。如果想在秋招顺利找到工作，就必须提高学习效率——一方面要尽快把基础知识过一遍，另一方面要尽快开始做项目。

简单来说，就是要加快学习节奏：快速学习、快速实践，通过项目不断提升自己的能力。

## 学习前的建议

因为时间非常宝贵，所以在很多时候不必再坚持手写所有代码。你可以把 AI 当成一个效率工具来使用——让 AI 帮你生成代码，从而节省大量时间。

包括后面自己做项目时，也可以让 AI 协助你完成代码编写。这样做的目的不是偷懒，而是把时间更多地用在理解思路、学习技术和推进项目上，而不是把大量时间花在一行一行慢慢敲代码上。

如果条件允许，建议你购买一个 Cursor 的会员。它在代码生成、代码解释以及项目开发辅助方面都会给你很大的帮助，能明显提升学习和开发效率。

推荐教程：[韩顺平学 Java](https://www.bilibili.com/video/BV1fh411y7R8/?spm_id_from=333.337.search-card.all.click)

## MySQL 数据库(两个月)

在企业开发中，大部分业务数据都是通过关系型数据库来存储的，因此数据库是后端开发人员必须掌握的核心技能。其中，MySQL 是目前最主流的数据库之一，也是面试中的重点考察内容，必须认真对待。

学习内容方面，需要掌握数据库的基本概念和 MySQL 的搭建方式，重点熟练编写 SQL 语句，包括 DDL（数据定义语言）、DML（数据操作语言）、DQL（数据查询语言）和 DCL（数据控制语言）。同时要理解各种约束，例如主键约束、外键约束、唯一约束和非空约束。索引是重中之重，需要掌握索引类型、索引原理以及索引优化思路；事务同样必须掌握，包括 ACID 特性和隔离级别。锁机制建议学习，例如行锁与表锁、乐观锁与悲观锁。此外，还需要具备设计数据库表的能力，包括理解三大范式、掌握表关联设计，以及具备一定的性能优化意识。

数据库是后端开发的基石，一定要学扎实。尤其是 SQL 语句编写能力和数据库表结构设计能力，这两个能力必须真正掌握。比如让你设计一个学生管理系统，你应该能够清晰地规划出需要哪些表，例如学生表、班级表；每张表需要哪些字段，以及字段的数据类型；表与表之间是什么关系，是一对多还是多对多。这些看似基础的问题，实际上很多同学在真正做项目时都会感到迷茫，所以一定要提前练熟练透。

推荐课程：[黑马程序员数据库](https://www.bilibili.com/video/BV1Y7411K7zz/?vd_source=87e9b301b5680ecec2487ab62d311d4a)

这门课程对于初学者来说已经完全够用，只要你能坚持从头到尾认真听完，并跟着动手练习，一定会有明显的提升。不要三天打鱼两天晒网，踏踏实实学完，收获绝对不会小。

在学习过程中，如果遇到自己不懂的内容，比如某些 数据结构，不要特意去找新的课程。

你可以直接问 AI，让它带着你一步步理解、掌握。因为不同的数据结构本身是相对独立的，AI 可以帮你快速搞明白核心概念和用法，而不用去浪费时间在零散的资料上。

重点还是要把 MySQL 学透、学会，这是你现在最需要完成的任务。其他零碎知识可以边学边用，以项目需求为主，不要偏离重点。

## Java Web(15天)

Java Web 阶段主要学习 Java 网页应用开发的基础内容，包括一点点前端基础知识、XML、JSON、Servlet、Filter、Listener、JSP、JSTL、Cookie 以及 Session 等核心内容。这一部分的意义，不是让你成为前端高手，而是为后续学习框架打好地基。因为很多框架本质上都是在这些技术之上一层层封装出来的，如果底层原理不清楚，后面学框架的时候就会很迷糊。

推荐课程：[黑马程序员 JavaWeb](https://www.bilibili.com/video/BV1Qf4y1T7Hx/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

跟着课程系统学一遍，理解原理并动手练习，后续学习 Spring 等框架时会轻松很多。

## 企业开发框架(两个月)

Java 之所以能够成为主流的企业级开发语言，很大一部分原因就在于它成熟而完善的框架生态。合理使用框架，不仅能够显著提升开发效率，还能增强项目的稳定性，并有效降低后期的维护成本。因此，企业级开发框架是后端开发中不可或缺的一部分，也是面试中的重点内容，一定要认真对待、深入掌握。

推荐学习资源如下：

[动力节点 Spring](https://www.bilibili.com/video/BV1Ft4y1g7Fb/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

[动力节点 SpringMVC](https://www.bilibili.com/video/BV1sC411L76f/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

[动力节点 MyBatis](https://www.bilibili.com/video/BV1JP4y1Z73S/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

[黑马程序员 Maven](https://www.bilibili.com/video/BV1Ah411S7ZE/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

[动力节点 Spring Boot 3](https://www.bilibili.com/video/BV1QNmZYrEp9/?spm_id_from=333.337.search-card.all.click&vd_source=87e9b301b5680ecec2487ab62d311d4a)

Spring 是一个 Java 轻量级应用框架，核心包括 IOC、AOP 和事务管理，能够简化企业级应用的开发。

SpringMVC 是 Java 的轻量级 Web 开发框架，基于 MVC 模式处理请求和响应，支持 Restful API，提供拦截器机制和灵活的配置方式，并有清晰的执行过程。

MyBatis 是数据访问框架，用于操作数据库进行增删改查，支持全局配置、动态 SQL、缓存管理，以及与其他框架的整合，还可以通过逆向工程快速生成代码。

SpringBoot 则是对 Spring 的进一步简化，能够快速搭建和开发应用，提高开发效率，常用功能包括常用注解、资源整合、高级特性和本地热部署。

Maven 是项目管理工具，用于项目构建、依赖管理、插件配置、子父工程管理以及多模块打包构建，还可以搭建 Nexus 私服以管理私有依赖。

按照顺序系统学习，结合动手实践，把基础框架理解清楚、把教程里面的项目跑通，比单纯“看过视频”重要得多。

## 苍穹外卖(两个月)

其实，上面这些技术如果你都系统学完了，找一份工作基本是没什么大问题的。但有一点必须清楚——只学习、不做题，在任何阶段都属于“假努力”。知识如果不通过实践巩固，很快就会变得模糊。

所以，你一定要自己完整做一个项目，把学过的东西真正用起来。

推荐项目：[苍穹外卖](https://www.bilibili.com/video/BV1TP411v7v6/)

这个项目在网上被不少人吐槽，甚至有点“烂大街”的感觉。但我想告诉你一个现实，大学生本来就很难接触到真正复杂的大型商业项目。你做项目的目的，也不是为了在面试时炫技，更不是为了显得多牛。

你做项目的唯一目的，是把学过的技术真正掌握住。它只是一个练习载体，仅此而已。这辈子你要做的项目多到数不过来，根本没必要在这里过度纠结“项目够不够高级”。能把一个项目完整跑通、独立写完、彻底理解，这才是最重要的。

## Git(三天)

Git 本质上只是一个工具型技术，它的核心作用就是帮助你管理代码版本。所以现阶段不需要研究得特别深入，只要会上传代码、会分支管理、会合并代码、会解决冲突，就已经够用了，其他高级玩法可以暂时不用纠结。

需要掌握的内容包括：区分 Git 和 GitHub 的作用，理解工作区的概念，掌握分支的使用，以及常见操作命令，例如代码提交、推送、拉取、回退、重置；熟悉分支操作流程，能够进行代码合并并解决冲突；了解标签的使用，掌握 cherry-pick 的基本用法，以及对 Git Flow 有一个整体认识。

学习的时候，每一个命令都要亲自敲一遍，至少形成一个大概印象，达到“需要时能够查阅并使用”的程度即可。平时建议多用 Git 管理自己的代码，多往 GitHub 上提交项目，用得多了，自然就熟练了。

推荐课程：[尚硅谷 Git 教程](https://www.bilibili.com/video/BV1vy4y1s7k6/)

这门课比较短，并且内容也比较简单，所以直接看视频学习即可。

## Linux(一周)

学习 Linux 的原因很简单——你的项目最终是要部署到 Linux 服务器上的。它本质上就是一个操作系统，所以现阶段不用把它想得特别复杂。你只需要会对文件进行增删改查、会安装软件、会把项目部署上去并成功运行，其实就已经够用了。

说实话，没有必要为了“学习 Linux”而单独投入太多时间啃理论教程。最有效的方式，是直接动手实践，自己买一台阿里云服务器，安装一个 Linux 系统，然后把自己的项目部署上去。只要你能成功部署并访问到，那这一步就算真正掌握了。

推荐教程：[韩顺平 Linux](https://www.bilibili.com/video/BV1Sv411r7vd/)

这个课程基于 CentOS 版本。你可能听说过 CentOS 已经停止维护，但问题不大。你只是去体验和熟悉 Linux 的使用方式，它本质上是工具。学的是基本指令和操作逻辑，而不是某个具体发行版。只要掌握了常用命令和部署流程，后面就算换成别的 Linux 系统，也能很快上手。

## 求职

如果上面这些内容你都学完了，恭喜你，至少可以应付一般的工作。不过我要先泼你一盆冷水：你目前掌握的这些技能，如果想找高薪工作，是不太可能支撑得住的。

秋招期间，你当然可以随便投简历，但心理预期不要太高。技术能力还不足以让你轻松拿到高薪岗位。但有面试还是要去试试，因为面试本身也是一种能力锻炼，它能帮你发现自己的不足，然后针对性地去补。

剩下的时间，就要把精力放在学习八股文上。由于你的基础比较薄弱，八股文对你来说可能会觉得很折磨。这个阶段不要想着死记硬背，因为背下来的内容很快就会忘。你需要做的是把八股文当作技术文章来学习，逐篇理解里面的原理和思路。

你可以让 AI 帮你逐篇分析、讲解，一点一点理解每篇内容，这也正好能弥补之前比较薄弱的基础。

还有一点建议：不要自己在网上零零散散找八股文资料，既浪费时间，又不系统。如果要学，最好找一套整理好的、体系化的资料。我个人比较推荐 [面试鸭](https://www.mianshiya.com/)，价格也就一百多块钱，但能帮你节省大量整理资料的时间。

## 最后的话

如果秋招没能找到工作，不要慌，记住还有春招。你还有大约半年多的时间。

这段时间最重要的就是总结秋招的经验，哪些知识点不足，哪些项目和技能没掌握，哪些面试环节不熟悉。然后针对性地去弥补缺口——缺什么就学什么，直击短板。

千万不要浪费时间去学那些网上特别长的教程或者复杂的学习路线，那些内容对你来说并不适合，只会拖慢你的进度。你的目标就是高效、精准地补齐能力缺口，为春招做准备。
